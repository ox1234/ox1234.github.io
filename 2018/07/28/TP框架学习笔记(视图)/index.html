TP框架学习笔记(视图)

模版定义

某个模块的模版文件是独立的，默认的模版文件定义规则是：

`视图目录/[模版主题]/控制器名/操作名+视图后缀

默认的视图目录是模块的View目录，框架默认的视图后缀为.html，

eg: Home模块下的User控制器的add方法对应的模版文件路径为：

`Application/Home/View/User/add.html

如果项目的视图不是View的话，可以通过'DEFAULT_V_LAYER' => 'Template'来更改

如果需要更改后缀，可以更改：

'TEMPL_TEMPLATE_SUFFIX' => 'tpl'

模版主题

如果某个模块要支持多个主题的话，可以使用模版主题功能

配置：'DEFAULT_THEME' => 'main'

模版赋值

如果要在模版中输出变量，需要在控制器中把变量传递给模版，系统提供了assign方法对变量赋值
